# 📝 Todo List App — Управление личной продуктивностью

Современный и отзывчивый веб-интерфейс для планирования, структурирования и контроля за ежедневными задачами. Приложение позволяет регистрировать персональные аккаунты, безопасно хранить списки дел в облачной базе данных, снабжать их развернутыми описаниями, редактировать на лету и отслеживать точное время создания и выполнения тасков.

---

## 🚀 Стек технологий

Приложение построено по клиент-серверной архитектуре (Monorepo):

* **Frontend:** Чистый JavaScript (Vanilla JS, ES6+), HTML5, CSS3 (Flexbox/Grid), архитектура компонентов и динамический рендеринг через DOM API.
* **Backend:** Node.js, Fastify (высокопроизводительный веб-фреймворк), JWT (JsonWebToken) для сессий авторизации, Bcrypt для надежного хеширования паролей.
* **База данных & BaaS:** Supabase (облачная СУБД PostgreSQL, управление сессиями через REST API).
* **Деплой / Хостинг:** Vercel / Render.

---

## ✨ Ключевые возможности и функции

1.  **Надежная авторизация:** Регистрация и вход пользователей с валидацией Email регулярными выражениями (RegEx) и шифрованием паролей.
2.  **Разделение контекста:** Каждый пользователь видит и управляет исключительно своими собственными задачами (Row Level Security / JWT Verification).
3.  **Полноценный CRUD:** Создание задач с названием и описанием, их редактирование во всплывающих формах и удаление через кастомное модальное окно.
4.  **Контроль времени:** Автоматическая фиксация системного времени закрытия задачи (`completed_at`) и вывод двух дат на карточке.
5.  **Интерфейс без прыжков:** Использование `min-width` и Flexbox для монолитного сохранения размеров кнопок и элементов при загрузках.

## 🔧 Локальный запуск проекта

### 1. Клонирование репозитория
```bash
git clone [https://github.com/Deloyzn/todo-list.git](https://github.com/Deloyzn/todo-list.git)
cd TodoList/
```

### 2. Установка зависимостей
Убедитесь, что у вас установлен Node.js, затем выполните:

```bash
npm install
```

### 3. Настройка переменных окружения
Создайте в корневой директории проекта файл .env и заполните его вашими ключами доступа к Supabase:

```code
VITE_SUPABASE_URL=ключи указанны в отчете
VITE_SUPABASE_PUBLISHABLE_KEY=ключи указанны в отчете
```

### 4. Запуск сервера
```bash
npm run dev
```
